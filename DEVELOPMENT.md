# 開發文檔 (DEVELOPMENT.md)

## 專案架構
此專案的架構由多個主要模組組成，每個模組都有其特定的功能。
- `src/`：原始程式碼位置。
- `tests/`：包含所有單元測試的目錄。
- `docs/`：相關文檔和使用手冊的目錄。

## 檔案結構
- `src/main.py`：主程序入口。
- `src/utils.py`：包含實用工具函式。
- `tests/test_main.py`：主程序的測試檔案。

## 如何修改功能
1. 確認你已經下載並安裝所有所需的依賴。
2. 使用你喜歡的編輯器打開對應的檔案。
3. 進行必要的修改。
4. 測試修改是否成功，確認所有單元測試都能通過。

## 偵錯提示
- 使用記錄器（logger）來幫助追蹤程式碼的執行流程。
- 當發生錯誤時，詳細檢查錯誤訊息，並記錄下來。
- 在有疑問的地方加入斷言語句，以確認假設的正確性。
